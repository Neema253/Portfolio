Contexte

Une plateforme d'écriture.

📖

L'idée du projet

  • Application inspirée de Wattpad
  • Chaque auteur gère sa bibliothèque privée
  • Les histoires sont découpées en chapitres ordonnés
  • Navigation fluide précédent / suivant entre chapitres
  • Expérience de lecture immersive style e-reader
  • Authentification Laravel Breeze requise
🏗️

Architecture MVC

  • StoryController — CRUD histoires + lecture
  • ChapterController — CRUD chapitres imbriqués
  • Routes protégées par middleware auth
  • Relations Eloquent : Story hasMany Chapter
  • Stockage des couvertures via storage:link
  • Validation serveur sur tous les formulaires

Fonctionnalités

Ce que fait Inkwell.

📚

Bibliothèque

  • Liste des histoires de l'auteur connecté
  • Stats : nb histoires, nb chapitres total
  • Couverture personnalisée (upload image)
  • Date de modification relative
  • Boutons Lire / Écrire / Modifier / Supprimer
  • État vide avec call-to-action
✍️

Éditeur

  • Toolbar : Gras, Italique, Souligné, H1, H2
  • Insertion citation (❝) et séparateur (—)
  • Compteur de mots en temps réel
  • Auto-save brouillon (localStorage, 30s)
  • Restauration du brouillon au rechargement
  • Mode plein écran + raccourci Ctrl+S
🌙

Lecture immersive

  • Barre de progression de lecture (scroll)
  • Table des matières fixe en sidebar
  • Navigation précédent / suivant par chapitre
  • Réglage taille de police (A- / A+)
  • Bascule Serif / Sans-serif
  • Thème clair / sombre (toggle)
🔐

Sécurité

  • Toutes les routes protégées par auth
  • Cloisonnement : chaque auteur voit uniquement ses histoires
  • abort(403) si accès à une histoire étrangère
  • Validation CSRF sur tous les formulaires
  • Validation serveur (title required, cover max 2Mo)
🗄️

Base de données

  • Table stories : title, description, cover, user_id
  • Table chapters : title, content, order, story_id
  • Relation hasMany / belongsTo
  • Ordre des chapitres géré automatiquement
  • Migrations versionnées
🎨

UX / Design

  • Thème sombre raffiné (noir profond + rouge + doré)
  • Animations fadeUp sur les cartes
  • Barre sticky navbar avec blur
  • Flash messages (succès / erreur)
  • Aperçu couverture avant upload
  • Design mobile responsive
✦ ✦ ✦
Points clés

Quelques exemples.

💾
Auto-save brouillon
L'éditeur sauvegarde automatiquement toutes les 30 secondes dans le localStorage. Au rechargement, il propose de restaurer le brouillon non enregistré.
📊
Barre de progression de lecture
En mode lecture, une barre rouge en haut de page indique l'avancement dans le chapitre au fil du scroll.
🔐
Cloisonnement par auteur
Chaque utilisateur n'accède qu'à ses propres histoires. Toute tentative d'accès à une histoire étrangère retourne un 403 Forbidden.
🌙
Réglages de lecture
Le lecteur peut ajuster la taille de police, basculer entre Serif et Sans-serif, et switcher entre thème clair et sombre.
