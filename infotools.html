Contexte

Un projet en quatre missions.

🖥️

Mission 1 — CRM lourd C# pour les managers

  • Application lourde C# / WinForms ou WPF
  • Gestion prospects, clients, contacts & rendez-vous
  • Gestion des produits, stocks et facturations
  • Authentification via Active Directory (AD)
  • Statistiques & graphiques exportables en PDF
  • Logs d'audit BDD (qui / quand / quoi / avant→après)
💼

Mission 2 — CRM Web pour les commerciaux

  • Authentification via Active Directory (LDAP / PHP)
  • Accès réservé aux commerciaux
  • Tableau de bord : rendez-vous à venir
  • Page produits de l'entreprise
  • Récapitulatif des achats par client
  • API REST sécurisée (Sanctum) + tests PHPUnit
🌐

Mission 3 — Site vitrine Laravel

  • Présentation de l'entreprise prestataire
  • Catalogue produits en base de données
  • Formulaire de demande de rendez-vous
  • Validation côté serveur (email, téléphone, message)
  • Journalisation des demandes dans audit_logs
  • Architecture MVC Laravel + vues Blade
📱

Mission 4 — Application CRM Android

  • Application mobile Xamarin / C#
  • CRUD complet des rendez-vous
  • Base de données SQLite locale
  • Notifications locales (AlarmManager + BroadcastReceiver)
  • Interface INotificationHelper via DependencyService
  • Rappel configurable : 15, 30 ou 60 minutes

Points clés

Ce qui rend ce projet technique.

🔐
Authentification Active Directory
Le CRM web authentifie les commerciaux via LDAP / Active Directory. Seuls les comptes du domaine Windows peuvent accéder à l'interface.
🛡️
Protection IDOR
Chaque commercial ne peut accéder qu'à ses propres rendez-vous. Toute tentative d'accès croisé entre comptes retourne un 403 Forbidden.
📋
Logs d'audit
Chaque création, modification ou suppression sur les tables critiques est journalisée avec l'identité de l'auteur, l'horodatage et les valeurs avant/après.
🔔
Notifications Android
L'app Android utilise AlarmManager + BroadcastReceiver pour déclencher des rappels configurables avant chaque rendez-vous.
