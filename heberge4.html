Heberge4

Plateforme d'hébergement web automatisé sous Linux Debian. Chaque utilisateur inscrit obtient automatiquement un compte Linux, un espace web Apache, une base de données MySQL, un enregistrement DNS Bind9 et un accès GLPI.

Contexte

Une plateforme d'hébergement.

🌐

L'idée du projet

  • Hébergement web gratuit et automatisé
  • Chaque utilisateur dispose d'un espace personnel
  • Inscription via interface PHP → tout se crée automatiquement
  • Accès GLPI intégré pour le support (ticketing)
  • Dashboard personnel de gestion du compte
  • Suppression complète du compte en un clic
🏗️

Architecture 3 serveurs

  • SrvWeb · 192.168.204.1 — Apache2, PHP, Bind9, interface
  • SrvBDD · 192.168.204.2 — MariaDB, base Projet & GLPI
  • SrvGLPI · 192.168.204.4 — GLPI 10.0.15, ticketing
  • Communication inter-serveurs via PDO MySQL
  • Scripts Bash exécutés en sudo depuis PHP
  • DNS zone heberge4.lan géré dynamiquement

Fonctionnement

Le cycle de vie d'un compte.

① Inscription — 7 étapes automatisées

1

Formulaire PHP

L'utilisateur saisit alias, nom, prénom, email et mot de passe. Données envoyées via PDO à la base Projet (SrvBDD) et simultanément à glpidb (SrvGLPI).

2

Lancement du script Bash

PHP exécute sudo /usr/local/bin/creation_utilisateur.sh $alias $mdp $nom $prenom $mail via exec().

3

Compte Linux

useradd -m -d /home/$ALIAS $ALIAS + echo "$ALIAS:$MDP" | chpasswd

4

Espace web

Création /home/$ALIAS/public_html/ avec index.html par défaut. Droits : chown -R $ALIAS:$ALIAS.

5

Base MySQL

CREATE DATABASE $ALIAS + GRANT ALL PRIVILEGES ON $ALIAS.* TO '$ALIAS'@'%'. Base isolée par utilisateur.

6

VirtualHost Apache

Copie template XXXX.conf, remplacement via sed, activation a2ensite + rechargement Apache.

7

DNS dynamique

echo "$ALIAS IN CNAME SrvWeb" >> /etc/bind/db.heberge4.lan + service bind9 reload.

② Suppression — 9 étapes de nettoyage

1

Déclenchement PHP

Clic "Supprimer le compte" → sudo /usr/local/bin/suppression_utilisateur.sh $alias.

2

9 nettoyages

BDD Projet · compte Linux (userdel) · /home/$ALIAS · utilisateur GLPI · a2dissite · conf Apache · rechargement Apache · entrée DNS · rechargement Bind9.

3

Session détruite

session_unset() + session_destroy() + redirection. Aucune trace ne subsiste.

Points clés

Ce qui rend ce projet technique.

🔗
PHP → Bash via exec() + sudo
L'interface PHP déclenche les scripts Bash système via exec("sudo /usr/local/bin/creation_utilisateur.sh " . escapeshellarg($alias)). Une approche qui couple le web et l'administration système.
🔄
Double synchronisation BDD
Chaque action met à jour simultanément deux bases sur deux serveurs distincts : Projet (192.168.204.2) et glpidb (192.168.204.4).
🌐
DNS dynamique avec Bind9
À chaque inscription, un enregistrement CNAME est ajouté dynamiquement dans /etc/bind/db.heberge4.lan avec un simple echo Bash.
🧹
Suppression en 9 étapes
Le script suppression_utilisateur.sh nettoie méthodiquement tout : BDD, Linux, home, GLPI, Apache, DNS. Aucune trace ne subsiste.
